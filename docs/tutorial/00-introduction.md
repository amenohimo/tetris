# はじめに — この教材の目的と進め方

## この教材で学ぶこと

あなたはこれから、**実際に動くブラウザゲーム（テトリス）** のソースコードを教材として、プログラミングの基礎を学びます。

テトリスは「ブラウザで動くゲーム」ですが、その裏では以下のような技術が使われています：

- **TypeScript**: プログラムに「型」という概念を導入し、バグを減らすプログラミング言語
- **Vite**: 開発中にコードの変更を即座に反映してくれるツール
- **Canvas API**: ブラウザ上で図形を描画する仕組み
- **ゲームループ**: ゲームを60fps（1秒間に60回）で更新し続ける仕組み

この教材では、これらを「テトリスという具体的なプログラム」を通じて学びます。「抽象的な説明だけ」や「退屈なサンプルコードだけ」ではありません。実際に動くコードを分解し、1行ずつ解説します。

## 対象読者

| 条件 | 説明 |
|------|------|
| プログラミング初心者 | 「Hello World」を書いたことがある程度で十分です |
| 独学中級者 | 他の言語は触ったけどTypeScriptは初めて、という人にも |
| 「なんとなく」を卒業したい人 | コードを写すだけでなく「なぜそう書くのか」を理解したい人 |

**必要な予備知識**:
- PCでファイルを作成・保存できる
- ブラウザ（Chrome / Firefox / Edge）を使ったことがある
- ターミナル（コマンドプロンプト / PowerShell / ターミナル）を起動したことがある（初めての場合は01で解説します）

## この教材の特徴

### 1. 「なぜそうするのか」を必ず説明する

プログラミングの学習で最も難しいのは「なぜこの書き方をするのか」を理解することです。この教材では、プロのプログラマが無意識にやっている判断を、すべて言語化します。

**例**: 「なぜゲームのロジック（Game.ts）と描画（Renderer.ts）を別のファイルに分けるのか？」

理由: ロジックと描画を分けると、「描画なしでロジックだけをテストできる」「将来、Canvasの代わりにWebGLを使う場合でも、Game.tsは変更しなくて済む」という利点があります。

### 2. 実際に動くプログラムを分解して解説する

この教材で解説するコードはすべて、`O:\_Programs\_tools\tetris\main\` に置かれている実際のテトリスゲームのものです。抽象的なサンプルコードではなく、実戦で使われているコードを通じて学べます。

### 3. プロジェクトの設計哲学を理解する

「動けばいい」というコードと「ちゃんと設計された」コードの違いは何でしょうか。このプロジェクトでは以下のような設計判断が行われています：

- Canvasで描画する（DOMではない、WebGLではない）
- TOML形式の設定ファイルを使う（JSONではない）
- 7-bag方式でピースを生成する（完全ランダムではない）
- DAS/ARR方式で入力処理する（OSのキーリピートに依存しない）

それぞれに「なぜその選択をしたのか」という理由があります。この教材ではそれらを全て解説します。

## 学習の進め方

1. **このファイル（00）** を読んで全体像をつかむ
2. **01** で開発環境をセットアップする。`npm install` と `npm run dev` を実行してみる
3. **02** でTypeScriptの基礎を学ぶ。わからなくなったらこのファイルに戻って復習する
4. **03** でプロジェクト全体の設計を理解する。ファイル構成と依存関係を把握する
5. **04** でテトリス自体のルールを理解する。遊びながら「この動きはコードでどう実装されているか」を考えてみる
6. **05〜12** で各モジュール（型定義、盤面、ピース、ランダマイザー、設定、ゲームエンジン、描画、入力）を1つずつ学ぶ
7. **13〜14** で全モジュールがどのように組み合わさって動いているかを理解する
8. **15** で学んだことを活かして改造に挑戦する

各ファイルの最後には **「自分で試してみよう」** という練習問題があります。必ず手を動かしてください。読むだけでは身につきません。
